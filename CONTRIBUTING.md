# Guía de Contribución

## Convención de Commits

Este proyecto usa commits semánticos:

- `feat:` Nueva funcionalidad
- `fix:` Corrección de bugs
- `refactor:` Reorganización de código
- `docs:` Cambios en documentación
- `test:` Añadir o modificar tests
- `chore:` Tareas de mantenimiento

Ejemplo:
```
feat: agregar endpoint de análisis por marca
fix: corregir cálculo de stock mínimo
refactor: separar lógica de negocio en services
```

## Flujo de trabajo

1. Crear rama para nueva funcionalidad
2. Hacer commits pequeños y descriptivos
3. Ejecutar tests antes de push
4. Actualizar documentación si es necesario
